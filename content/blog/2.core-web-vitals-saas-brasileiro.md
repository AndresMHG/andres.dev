# Core Web Vitals para SaaS brasileiro

Em março de 2024 o Google substituiu o FID pelo **INP (Interaction to Next Paint)**, e mudou o jogo para todo SaaS com formulários, dashboards e interações pesadas.

## Os 3 que importam hoje

- **LCP** ≤ 2.5s — Largest Contentful Paint
- **INP** ≤ 200ms — substituiu o FID em março/2024
- **CLS** ≤ 0.1   — Cumulative Layout Shift

## INP: o assassino silencioso do seu SaaS

Diferente do FID (que mediu só a primeira interação), o INP mede a **pior** interação da sessão inteira. Se você tem um modal pesado, um autocomplete laggy, ou um botão que dispara muito trabalho síncrono — o INP te pega.

Como medir em produção: `web-vitals` library + Datadog RUM ou Sentry Performance.

## Como priorizar quando o backlog é infinito

Use a matriz **impacto × esforço**:

1. Identifique a métrica pior posicionada vs o threshold
2. Para essa métrica, liste as 10 maiores causas (Lighthouse + WebPageTest)
3. Ordene por impacto estimado / horas de dev
4. Faça os 3 primeiros antes de tocar em qualquer outra coisa
