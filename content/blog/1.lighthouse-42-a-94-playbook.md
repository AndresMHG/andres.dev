# Lighthouse 42 → 94: o playbook completo

Esse é o primeiro post de uma série sobre o que aprendi liderando uma migração Nuxt 2 → 3 em produção, com foco em performance real (não performance só pra Lighthouse).

## TL;DR

| Métrica                    | Antes  | Depois | Δ      |
|----------------------------|--------|--------|--------|
| Lighthouse Performance     | 42     | 94     | +124%  |
| LCP                        | 4.8s   | 1.6s   | -67%   |
| Bundle JS (gzipped)        | 820 kb | 310 kb | -62%   |
| Tráfego orgânico (3 meses) | base   | +38%   | —      |

## O que mais moveu a agulha

1. **Code splitting agressivo por rota** + lazy load de componentes pesados
2. **`<NuxtImg>` com `width`/`height` + lazy** para eliminar CLS
3. **Self-host das fonts** com `preload` (eliminar render-blocking do Google Fonts)
4. **SSR + payload extraction** ao invés de SPA puro
5. **Critical CSS inline** (`features.inlineStyles`)

Cada um desses tópicos vira um post próprio nas próximas semanas.
